# Git Tags 檢視器 - Electron 桌面版

將原本的 Shell 腳本 `generate-tags-viewer.sh` 完全整合到 Electron 桌面應用程式中，提供圖形化介面來設定和生成 Git 標籤檢視器。

## 功能特色

### 🖥️ 圖形化介面
- 友善的桌面應用程式介面
- 即時的 Git 專案驗證
- 可自訂的標籤分組前綴
- 彈性的顯示設定

### 🏷️ 完整的標籤分析
- 自動分析 Git 標籤並按前綴分組
- 支援版本間差異檢視（當 commits 設為 0 時）
- 可設定每組顯示的標籤數量
- 可設定每個標籤顯示的提交數量

### 🔍 進階搜尋功能
- 即時搜尋標籤名稱和提交訊息
- 高亮顯示搜尋結果
- 分組統計資訊

### ⚡ 一鍵生成
- 按下按鈕即可生成檢視器
- 自動開啟新視窗顯示結果
- 取消了 .env 文件依賴

## 使用方式

### 1. 設定專案路徑
- 點擊「瀏覽」按鈕選擇包含 .git 的專案目錄
- 或直接輸入專案路徑
- 系統會自動驗證是否為有效的 Git 專案

### 2. 調整顯示設定
- **每組標籤數**: 每個分組最多顯示幾個標籤（預設 10）
- **每標籤提交數**: 每個標籤顯示幾個提交（預設 5）
  - 設為 0: 顯示版本間的所有差異提交
  - 設為其他數字: 顯示該標籤的最新 N 個提交

### 3. 設定標籤分組前綴
- 預設包含 prd-v、uat-v、v1.0.、preview-v、lab- 等分組
- 可以新增、移除或修改分組前綴
- 系統會按照內建優先序排序分組

### 4. 生成檢視器
- 點擊「生成標籤檢視器」按鈕
- 系統會分析 Git 標籤並生成檢視器
- 完成後會自動開啟新視窗顯示結果

## 核心功能實作

### Git 操作邏輯
- 使用 `simple-git` 套件處理 Git 操作
- 支援標籤列表、提交歷史、版本間差異等功能
- 完全移植原 Shell 腳本的分組和排序邏輯

### 安全性設計
- 使用 `contextIsolation` 和 `preload.js` 確保安全性
- 主程序和渲染程序間使用 IPC 通訊
- 不直接暴露 Node.js API 給前端

### 檢視器整合
- 將生成的 HTML 檢視器直接在新視窗中顯示
- 保留原有的搜尋、展開收合、統計等功能
- 美化的視覺設計和動畫效果

## 與原 Shell 腳本的差異

### 優勢
- ✅ 圖形化設定介面，不需要手動編輯 .env 文件
- ✅ 一鍵生成，無需記憶命令列參數
- ✅ 即時驗證和錯誤提示
- ✅ 跨平台桌面應用程式
- ✅ 可建置成獨立執行檔

### 相容性
- ✅ 完全相容原有的標籤分組邏輯
- ✅ 支援所有原有的顯示模式
- ✅ 保留搜尋和統計功能
- ✅ 維持相同的檢視器使用體驗

## 系統需求

- Node.js 16 或以上版本
- Git 已安裝並可在命令列使用
- macOS、Windows 或 Linux 作業系統

## 開發說明

這個應用程式採用 Electron 的最新安全實踐：
- 關閉 `nodeIntegration`
- 啟用 `contextIsolation`
- 使用 `preload.js` 安全暴露 API
- 主程序和渲染程序分離

如需客製化或擴充功能，請參考各檔案的註解說明。
